== Delivery Discovery Alignment

Документация должна начинаться с delivery discovery: сначала фиксируются problem statement, users/workflows, scope/non-goals, measurable success criteria, data/integration assumptions, risks, pilot gates and release gates. Архитектурные документы и ADR должны ссылаться на эти discovery decisions, а не заменять их.

Для инициатив с AI/LLM/RAG используйте шаблоны из xref:ml-llm-delivery-pipeline/01-prd-template.adoc[ML LLM Delivery Pipeline]:

* xref:ml-llm-delivery-pipeline/01-prd-template.adoc[01 PRD Template] для service PRD / PRD-lite.
* xref:ml-llm-delivery-pipeline/05-evaluation-plan-template.adoc[05 Evaluation Plan Template] для offline eval, release gates and regression policy.
* xref:ml-llm-delivery-pipeline/06-delivery-plan-template.adoc[06 Delivery Plan Template] для workstreams, owners, dependencies, rollout and delivery gates.

== Building the Documentation Site Locally

The docs are published as an Antora site. The playbook lives at the repository root
(`antora-playbook.yml`).

Generate the site:

[source,bash]
----
npx antora generate antora-playbook.yml
----

Output goes to `build/site/index.html`. Open it directly in a browser, or serve it:

[source,bash]
----
npx http-server build/site
----

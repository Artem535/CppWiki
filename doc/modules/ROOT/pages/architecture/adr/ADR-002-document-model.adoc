= ADR-002 — Document Model

== Status

Accepted

== Date

2026-06-11

'''

== Context

The previous PRD used Djot as the primary storage format.

That created a pipeline:

----
Djot
    ↓
AST
    ↓
HTML / ADF / Confluence Storage
----

This made sense for a markup-first wiki, but not for a block-based Confluence/Notion-like product.

'''

== Decision

Use a structured JSON Block Document as the source of truth.

Djot, Markdown, HTML, ADF and Confluence Storage Format are import/export formats.

'''

== Consequences

=== Positive

* Better fit for block editor.
* Stable block IDs.
* Easier comments and future collaboration.
* Easier plugin block representation.
* Better Confluence mapping.

=== Negative

* No plain-text-first storage.
* Requires schema migration system.
* Requires custom export/import converters.

'''

== Example

[source,json]
----
{
  "schema_version": "1.0",
  "blocks": [
    {
      "id": "block-uuid",
      "type": "paragraph",
      "props": {},
      "content": [],
      "children": []
    }
  ]
}
----

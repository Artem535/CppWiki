= ADR-005 — Rendering Pipeline

== Status

Accepted

== Date

2026-06-11

'''

== Context

Block editors render content interactively for editing.

However, editor rendering is not sufficient for:

* PDF export;
* Confluence export;
* search indexing;
* read-only mode;
* security sanitization;
* deterministic output.

'''

== Decision

Separate rendering into two layers:

. Editor Render.
. Canonical Render.

Editor Render belongs to BlockNote.

Canonical Render belongs to C++ RenderService.

'''

== Consequences

=== Positive

* One authoritative export pipeline.
* Plugin rendering is centralized.
* Search and export are consistent.
* Security sanitization is easier.

=== Negative

* Some rendering logic exists in two forms.
* Plugin authors may need to provide editor schema and canonical render behavior.

'''

== Pipeline

----
Document JSON
    ↓
Internal AST
    ↓
C++ RenderService
    ↓
WASM Plugin Renderers
    ↓
HTML / ADF / Confluence Storage / Search Text
----
